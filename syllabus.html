Course Syllabus (14 Weeks)

Phase 1: Brownfield (Weeks 1-7)

Module 1: Context & Efficiency

Week 1: Staying under context limits with markdown systems and retrieval budgets Lab theme: prompt contracts + context budgeting (token-first thinking)

Week 2: Token optimization for large codebases Lab theme: chunking strategy + query design + evaluation rubric

Module 2: Architectural State

Week 3: Memory management (short-term vs long-term) Lab theme: AI memory artifacts + consistent update rules

Week 4: Caching strategies + managing technical debt with AI assistance Lab theme: decision logs + debt tracking + reuse patterns

Module 3: Extended Tools

Week 5: MCP-first navigation and refactoring planning Lab theme: tool contracts + traceable edits

Week 6: Worktrees for safe parallel refactors in multi-repo legacy systems Lab theme: branching safety + rollback discipline

Week 7: Brownfield studio (end-to-end refactor with verification) Lab theme: from code map -> patch -> tests -> rollout plan

Phase 2: Greenfield (Weeks 8-14)

Module 4: Foundations & Skills

Week 8: AI-ready project architecture from scratch Lab theme: docs + structure + test scaffolding that makes AI useful

Week 9: Core engineering skills for applied AI (evaluation as a habit) Lab theme: prompt suites + acceptance checks + regression tests

Module 5: Extensibility

Week 10: Developing plugins to extend LLM capabilities Lab theme: tool/plugin contracts + versioning + safety boundaries

Week 11: Integrating plugins into real workflows Lab theme: orchestration + fallback behavior + measurement

Module 6: Autonomous Agents

Week 12: Single-agent systems for specialized development tasks Lab theme: tool-using loop + guardrails + structured outputs

Week 13: Multi-agent systems (coordination, handoffs, arbitration) Lab theme: role design + conflict resolution + trace logs

Module 7: Advanced Workflows (Final Integration)

Week 14: Orchestrating multiple agents + complex MCP integrations for a production-ready Greenfield MVP Lab theme: end-to-end capstone workflow with verification and handoff docs

What you submit each week

  • A lab deliverable (document, checklist, patch plan, or prototype artifact)
  • A short reflection: what worked, what failed, and what you would change next time

Final capstone outcome

At the end of Week 14, you will have a production-oriented Greenfield MVP plan and workflow artifacts that demonstrate:

  • context efficiency,
  • architectural state management,
  • tool-first navigation and safe refactoring,
  • plugin extensibility,
  • agent orchestration with verification.
