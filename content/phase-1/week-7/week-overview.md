## Learning outcomes

By the end of this week, students should be able to:

1. **Describe** failure modes at **monorepo scale** in **Canvas LMS**: unclear boundaries between `app/`, `ui/`, `packages/`, `gems/`, and build steps.
2. **Use** AI assistance to draft a **subsystem integration map** for an approved Canvas feature or fix (dependencies, build/runtime touchpoints).
3. **Locate** where contract truth lives for relevant APIs, plugin hooks, or external integrations (per Canvas docs and code pointers).
4. **Produce** a refactor or migration **plan as text**: scope, milestones, risks, and test strategy—without hand-authoring implementation.
5. **Summarize** the plan for stakeholders (scope, unknowns, next steps) at an appropriate level of abstraction.
6. **Participate** in peer critique to find gaps in others’ plans and **deliver** a Phase 1 portfolio checkpoint: Canvas-scoped plan plus an updated **context pack**.
