# Lab · Phase 1 Week 4 (Calendar week 4)
## Caching strategies & staying aligned with reality

**Codebase:** [Canvas LMS](https://github.com/instructure/canvas-lms) · [`../../brownfield-project.md`](../../brownfield-project.md)

### Goal
Design a lightweight **knowledge cache** (not necessarily RAG)—rules for write/read/invalidate.

### Steps

1. Choose one repeating question people ask about **Canvas** (e.g., how to run tests, build front-end assets, or start Docker)—ground answers in `README`, `doc/`, or `docker-compose/`.
2. Have AI propose: canonical doc location, summary card, and links to sources.
3. Define **invalidate on** events (release, config change, etc.).
4. Role-play: new teammate uses only your cache card—note gaps.

### Definition of done
One-page policy + one worked summary card.
