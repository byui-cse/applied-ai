# Lab · Phase 1 Week 3 (Calendar week 3)
## Architectural state & project memory

**Codebase:** [Canvas LMS](https://github.com/instructure/canvas-lms) · [`../../brownfield-project.md`](../../brownfield-project.md)

### Goal
Create a **Debt Radar** and a **Last Verified** ritual.

### Steps

1. AI generates candidate debt items tagged: reliability, security, maintainability, performance.
2. You force each item to cite **evidence** (path, test name, metric, or ticket)—drop uncited items.
3. Add a **Last verified** field per section of your architecture notes.
4. Optional: simulate onboarding—can a classmate answer 'top 3 risks' from your pack only?

### Definition of done
10–20 high-signal items max; each has evidence or is marked 'unverified'.
