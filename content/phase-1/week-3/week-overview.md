**Brownfield codebase:** [Canvas LMS](https://github.com/instructure/canvas-lms)

## Learning outcomes

By the end of this week, students should be able to:

1. **Distinguish** chat memory, git history, and durable project documentation—and explain when each is trustworthy.
2. **Describe** when to re-ground AI-assisted work (e.g., after merges, incidents, refactors) and what must be refreshed in docs.
3. **Mitigate** stale architecture notes using dated **last verified** metadata and explicit uncertainty.
4. **Prompt** an AI to produce a structured **debt radar** across coupling, tests, operations, and similar dimensions.
5. **Prioritize** debt items using impact × uncertainty and **avoid** unprioritized laundry lists.
6. **Specify** a human verification gate: what evidence is required before treating a debt item as confirmed.
