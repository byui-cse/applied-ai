## Learning outcomes

By the end of this week, students should be able to:

1. **Explain** the MCP mental model at a high level (e.g., tools, resources, prompts—exact vocabulary may vary by host).
2. **Analyze** trust boundaries: what data an MCP server can access and what must stay off-limits.
3. **Integrate** MCP-backed tooling into an IDE workflow for **Canvas LMS** exploration without losing track of provenance.
4. **Identify** tasks in a repo the size of **Canvas** that benefit from MCP (cross-cutting search, schema/doc bridges, etc.).
5. **Propose** guardrails for tool use (logging, allowlists, dry-runs) appropriate to a campus or org policy.
6. **Document** an **MCP inventory**: enabled servers, exposed surfaces, and explicit risk notes.
