**Module:** 4 — Foundations & skills  

## Learning outcomes

By the end of this week, students should be able to:

1. **Encode** repeatable team habits for the model using the host’s mechanism (rules, skills, memories, or equivalent).
2. **Decide** when to centralize guidance versus when to scope instructions per task or feature.
3. **Operate** a review cadence to prune stale or contradictory rules.
4. **Critique** AI-generated tests for value versus brittle coverage—without personally writing implementation code.
5. **Apply** review checklists (security, accessibility, performance, etc.) as **gates** on AI output.
6. **Produce** a **rules/skills pack v1** and a reusable **review checklist** for the greenfield MVP.
