**Module:** 6 — Autonomous agents  

## Learning outcomes

By the end of this week, students should be able to:

1. **Describe** single-agent role patterns (e.g., planner, executor, reviewer) and how one agent can switch hats with explicit prompts.
2. **Define** stop conditions and budgets (time, tokens, files touched) for agent runs.
3. **Insert** human-in-the-loop checkpoints where verification is mandatory.
4. **Decide** when to spin an agent versus staying in interactive chat for a given chore.
5. **List** artifacts agents must leave behind (logs, summaries, tickets) for auditability.
6. **Author** a **single-agent runbook** for one recurring MVP chore, including evaluation criteria for outputs.
