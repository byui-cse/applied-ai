# Week 8: Foundations & Skills - Build AI-Ready Architecture from Scratch

## Learning objectives
- Scaffold a repository so LLM workflows are easier and safer
- Create documentation and test scaffolding that reduces prompt ambiguity
- Establish conventions for future tool integrations (plugins and agents)

## Key ideas
- AI-ready architecture includes structure for: docs, examples, tests, and stable interfaces
- Consistency beats novelty: predictable folder patterns reduce cognitive load
- Good evaluation starts with acceptance criteria you can test

## Lab: Repo scaffolding + "AI handbook"
1. Create `lab/week-8/ai-handbook.md` and include:
   - project overview (what it does)
   - architecture map (components and boundaries)
   - "how to run" instructions (commands, env vars placeholders)
   - how to add a feature (a checklist)
2. Create `lab/week-8/acceptance-criteria.md` for a small feature you will implement later in the course
3. Add a test plan section: what unit tests and integration checks you expect
4. Write a short "prompt contract" for future AI work that references your handbook (inputs/outputs + required verification)

## Submission checklist
- `ai-handbook.md`
- `acceptance-criteria.md`
- Prompt contract (short, reusable)

## Reflection questions
- What documentation would have saved you time last week in brownfield work?
- Which conventions will you keep stable as the project grows?
