# Week 9 Lab: Markdown Memory (Prompt Suite + Evaluation Rubric)

## Prompt suite structure (template)
Write 3 representative tasks.

```markdown

### Task brief
<one paragraph>

### Inputs you will provide
<files/config snippets/constraints>

### Output format
<what the model must produce>

### Acceptance properties
<what must be true>

## Task 2
...

## Task 3
...
```

## Evaluation rubric template
Create 4-6 criteria and explain how you check each one.

```markdown
## Evaluation rubric (4-6 criteria)
1. <criterion>
   - How to check:
   - What “good” looks like:
2. <criterion>
   - ...
```

## Dry evaluation (template)
Pick one hypothetical model response and score it.

```markdown
## Dry evaluation: <Task name>
- Rubric scores:
  - <criterion>: <score + note>
- Evidence from the response:
  - <quotes / paraphrases>
- Summary:
  - <what is correct, what fails, what to change in the prompt>
```

## Deliverables
- `prompt-suite.md` (or separate files, but fill everything)
- `evaluation-rubric.md`
- 1-2 sentences on how you will run regression checks later
