**Module:** 7 — Advanced workflows (final integration)  

## Learning outcomes

By the end of this week, students should be able to:

1. **Reason** about multiple MCP servers coexisting: naming, ownership boundaries, and overlap between tools.
2. **Prioritize** tool and agent actions under load using simple, defensible heuristics.
3. **Maintain** incident-ready documentation when AI-assisted changes land in the system (what changed, how to roll back, how to verify).
4. **Apply** a production checklist (auth, secrets, logging, backups, etc.) to the greenfield MVP at an appropriate level of rigor.
5. **Deliver** a coherent **demo narrative**: problem → MVP → limitations → next steps.
6. **Assemble** a **Phase 2 final package** per rubric: prompts log, architecture snapshot, rules pack, and reproducible workflow notes for a subset of the work.
