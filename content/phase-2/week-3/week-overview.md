**Module:** 5 — Extensibility  

## Learning outcomes

By the end of this week, students should be able to:

1. **Compare** common plugin shapes: callable tools/functions, UI extensions, and bridges to external systems (e.g., MCP).
2. **Analyze** trust requirements: permissions, sandboxing, and secret handling for extensibility.
3. **Judge** when plugins improve UX versus add clutter or attack surface.
4. **Specify** stable interfaces for a plugin surface: schemas, versioning, and deprecation expectations.
5. **Draft** (with AI assistance) a plugin specification covering inputs, outputs, and error cases.
6. **Deliver** a **plugin spec v0**, including a JSON-schema (or equivalent) sketch produced via AI collaboration.
